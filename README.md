# متجري — نظام إدارة المبيعات والمخزون 🛍️

تطبيق أندرويد **يعمل بدون إنترنت** لإدارة عمل تجاري متكامل: مخزون، مبيعات، موردون، عملاء، أرباح، وتقارير PDF. بهوية احترافية عصرية (زمردي مالي على خلفية فاتحة نظيفة) مع شعار حقيبة التسوق.

---

## ✨ المزايا

- **لوحة تحكم** بأرقام كبيرة وعدّاد أرباح متحرك ورسم بياني.
- **المخزون**: صور، تصنيف، لون، مقاس، كمية، أسعار، حالة، بحث وفلاتر.
- **المبيعات**: فواتير، سلة، حالة دفع، حساب ربح تلقائي. **وتظهر صور المنتجات المشتراة داخل ملف كل عميل.**
- **الموردون والعملاء**: أرصدة ومستحقات، دفعات، اتصال وواتساب، **اختيار من جهات الاتصال (يعمل بعد التثبيت).**
- **العملات**: ريال يمني + سعودي مع سعر صرف يدوي.
- **تقارير PDF حقيقية** تُحفظ كملف، وتصدير CSV (Excel).
- **مزامنة عبر الإنترنت** مع برنامج الكمبيوتر (server.js).
- مصروفات، حماية PIN، نسخ احتياطي، سجل عمليات، وضع داكن/فاتح، RTL كامل.

## 📱 بناء ملف APK عبر GitHub Actions

### ⚠️ تنبيه مهم — مجلد `.github` المخفي
عند الرفع بالسحب من المتصفح **لا يُرفع مجلد `.github`** (لأنه مخفي)، وبدونه لا يعمل البناء. كذلك تأكد أن `index.html` و`vendor/` داخل مجلد **www/**. الأضمن استخدام Git:

```bash
cd matjari
git init
git add .
git commit -m "متجري — الإصدار 1.0"
git branch -M main
git remote add origin https://github.com/USERNAME/REPO.git
git push -u origin main
```

> إن رفعت من المتصفح: أنشئ ملف الـ workflow يدويًا عبر **Add file → Create new file** واكتب المسار `.github/workflows/build-apk.yml`، وتأكد أن ملفات `vendor/jspdf.umd.min.js` و`vendor/html2canvas.min.js` مرفوعة داخل `www/vendor/`.

### تشغيل البناء وتنزيل الـ APK
يبدأ تلقائيًا بعد الرفع، أو من تبويب **Actions → Run workflow**. بعد ~5–8 دقائق: **Actions ← آخر تشغيل ← Artifacts ← matjari-apk** ← فُكّ الضغط ← ثبّت `app-debug.apk`.

> البناء يولّد أيقونة التطبيق (حقيبة التسوق الذهبية) تلقائيًا من `resources/icon.png`، ويضيف صلاحية جهات الاتصال.

## 🖥️ ربط التطبيق ببرنامج الكمبيوتر (المزامنة عبر الإنترنت)

`server.js` هو برنامج كمبيوتر بسيط يستقبل بيانات التطبيق ويعرضها في لوحة، **بدون أي مكتبات خارجية** (يحتاج Node.js فقط).

**على الكمبيوتر:**
```bash
node server.js
```
سيطبع عنوانه. افتح `http://localhost:4321` لرؤية اللوحة.

**في تطبيق الجوال:** المزيد ← المزامنة عبر الإنترنت ← اكتب عنوان الكمبيوتر (مثل `http://192.168.1.5:4321`) وكودًا للمتجر، ثم **رفع للسحابة**. فعّل «مزامنة تلقائية» ليُرفع كل تغيير تلقائيًا، أو استخدم **جلب من السحابة** على جهاز آخر بنفس الكود.

- **نفس شبكة الواي‑فاي:** اكتب عنوان الجهاز المحلي (تجده بـ `ipconfig` على ويندوز أو `ifconfig` على ماك/لينكس). التطبيق المُجمّع يسمح بالاتصال المحلي تلقائيًا.
- **من أي مكان عبر الإنترنت:** شغّل نفقًا مجانيًا أمام الخادم، مثل **Cloudflare Tunnel** أو **ngrok** (`ngrok http 4321`)، واكتب الرابط الذي يعطيك إياه (https) في التطبيق.

> البيانات تُحفظ في مجلد `data/` بجانب `server.js`، ملف JSON لكل متجر.

## 📄 أين تُحفظ الملفات (PDF / النسخة الاحتياطية / CSV)؟

- **داخل التطبيق المُجمّع (الجوال):** يُحفظ في مجلد **Documents**، ثم تظهر نافذة مشاركة للحفظ في «الملفات» أو الإرسال عبر واتساب.
- **في المتصفح:** يُنزّل إلى مجلد **التنزيلات (Downloads)**.

كل الملفات (تقارير PDF، النسخة الاحتياطية JSON، تصدير CSV) تُحفظ بنفس الطريقة عبر نظام ملفات الجهاز الأصلي.

التقارير والفواتير تُولَّد كصورة عالية الدقة داخل PDF (تدعم العربية بالكامل) عبر مكتبتي jsPDF و html2canvas المضمّنتين للعمل دون إنترنت.
